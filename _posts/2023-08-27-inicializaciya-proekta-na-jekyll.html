Почему Jekyll

Jekyll - это генератор статических сайтов, который позволяет создавать веб-сайты, состоящие из статических HTML-файлов, без необходимости использования серверных технологий или баз данных. Благодаря чему вместо покупки даже самомального виртуального хостинга можно использовать бесплатный GitHub Pages.

По функциональности он практически не уступает тем же самым CMS системам для введения блога. Он на много производительней и прост в освоении. Идеальный вариант для создания новостных порталов, блогов и портфолио. В частности данный сайт написан на Jekyll.

Базовая установка

Полная инструкция по установке Jekyll описана на официальном сайте, здесь же я приведу пример установки на дистрибутив Pop!_OS.

Установите Ruby и других необходимых зависимостей.

{% include component/code.html lang='bash' content='sudo apt-get install ruby-full build-essential zlib1g-dev' %}

Настройки для учетной записи пользователя. Введите ниже представленные команды попорядку.

{% capture code %} echo '# Install Ruby Gems to ~/gems' >> ~/.bashrc echo 'export GEM_HOME="$HOME/gems"' >> ~/.bashrc echo 'export PATH="$HOME/gems/bin:$PATH"' >> ~/.bashrc source ~/.bashrc {% endcapture %} {% include component/code.html lang='bash' content=code %}

Установка Jekyll и Bundler.

{% include component/code.html lang='bash' content='gem install jekyll bundler' %}

Создание проекта

Полную инструкцию по создании проекта Jekyll описана на официальном сайте.

Создать jekyll проект в каталоге myblog.

{% include component/code.html lang='bash' content='jekyll new myblog' %}

Переходим в каталог myblog.

{% include component/code.html lang='bash' content='cd myblog' %}

Устанавливаем расширения прописанные в файле Gemfile.

{% include component/code.html lang='bash' content='bundle install' %}

Запускаем проект.

{% include component/code.html lang='bash' content='bundle exec jekyll serve' %}

Переходим в браузере на виртуальный сервер http://localhost:4000.
